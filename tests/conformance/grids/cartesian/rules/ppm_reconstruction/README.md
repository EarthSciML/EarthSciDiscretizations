# ppm_reconstruction — rule-eval cross-binding fixture

Verifies that each binding's PPM reconstruction evaluator (TypeScript landed
in dsc-rys; Python and Rust siblings in flight) reproduces the closed-form
edge values and parabola samples to within the declared tolerance when fed
the same on-disk multi-stencil rule and the same cell-averaged input field.

## Layout

- `fixtures.json` — fixture spec (grid, manufactured solution, query cells,
  parabola sub-cell coordinates, tolerance). Shared across bindings.
- `golden.json` — closed-form edge values + parabola samples, regenerated by
  `regenerate_golden.mjs`.
- `regenerate_golden.mjs` — binding-independent regenerator that writes
  `golden.json` directly from the closed-form CW84 formulas the rule
  (`discretizations/finite_volume/ppm_reconstruction.json`) encodes.

## Comparison protocol

For each query cell `i` on a 16-cell uniform Cartesian grid with periodic
boundaries, each binding:

1. Computes cell averages of the manufactured solution
   `f(x) = sin(2πx) + 0.3·cos(4πx)` from its closed-form antiderivative
   `F(x) = -cos(2πx)/(2π) + 0.3·sin(4πx)/(4π)`.
2. Applies the rule's `q_left_edge` and `q_right_edge` sub-stencils via its
   PPM evaluator.
3. Samples the cell parabola at `xi ∈ {0.1, 0.3, 0.5, 0.7, 0.9}`.

Results are compared against `golden.json` using the relative tolerance in
`fixtures.json.tolerance.relative` (currently `1e-13`).

## Regenerating

```bash
node tests/conformance/grids/cartesian/rules/ppm_reconstruction/regenerate_golden.mjs
```

The regenerator does NOT use any binding's AST evaluator — it inlines the
closed-form coefficients (CW84 eq. 1.6 for the edges; eqs. 1.5/1.7/1.10 for
the parabola). This keeps the golden binding-independent so each evaluator
is checked against an external reference rather than itself.
