# centered_2nd_uniform_latlon — rule-eval cross-binding fixture

Verifies that the per-binding rule evaluators (Julia `eval_coeff`, the
TypeScript `rules.evaluate` introduced in dsc-e1g, and the Python +
Rust siblings under dsc-ve1 / dsc-k86) produce numerically identical
coefficient values when fed the same ESS-emitted ExpressionNode AST and
the same bindings.

## Layout

- `fixtures.json` — fixture spec (grid opts, query points, stencil order,
  tolerance), shared across bindings
- `golden.json` — closed-form coefficient values, regenerated by
  `regenerate_golden.mjs`
- `regenerate_golden.mjs` — binding-independent regenerator that writes
  `golden.json` directly from the closed-form coefficient formulas in
  `discretizations/finite_difference/centered_2nd_uniform_latlon.json`

## Comparison protocol

For each query point `(j, i)` on a small `regular` lat-lon grid, each
binding evaluates the rule's four stencil coefficient ASTs against the
bindings:

- `R` — sphere radius
- `dlon = 2π / nlon`
- `dlat = π / nlat`
- `cos_lat = cos(lat at cell center j, i)`

Results are compared against `golden.json` using a relative tolerance of
`1e-14` (matches the GRIDS_API §4.2 default for closed-form geometry).

## Regenerating

```bash
node tests/conformance/grids/latlon/rules/centered_2nd_uniform_latlon/regenerate_golden.mjs
```

The regenerator does NOT use any binding's AST evaluator — it computes
coefficients directly from the closed-form formulas the rule encodes:

- `lon, offset ±1`: `±1 / (2 * R * cos_lat * dlon)`
- `lat, offset ±1`: `±1 / (2 * R * dlat)`

This keeps the golden binding-independent so each evaluator is checked
against an external reference rather than itself.
