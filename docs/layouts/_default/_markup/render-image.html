{{- /* Markdown image render hook. Hugo does NOT prefix the site's baseURL onto root-relative ("/foo") image destinations in markdown content. When the site is published under a path prefix (e.g. https://earthsciml.github.io/EarthSciDiscretizations/), an emitted ![](/plots/x.png) renders as , which the browser resolves at the apex (earthsciml.github.io/plots/x.png → 404) instead of inside the project prefix (earthsciml.github.io/EarthSciDiscretizations/plots/x.png). Generated rule pages emit root-relative plot refs by design — the asset is copied to docs/static/plots//. Run every root-relative destination through relURL so the prefix is applied, while leaving protocol-relative ("//host/x"), absolute ("https://..."), and page-relative ("foo.png") destinations untouched. */ -}} {{- $dest := .Destination -}} {{- if and (hasPrefix $dest "/") (not (hasPrefix $dest "//")) -}} {{- /* relURL leaves leading-slash paths unchanged; trim the slash first so the baseURL path component is prepended. */ -}} {{- $dest = strings.TrimPrefix "/" $dest | relURL -}} {{- end -}}
